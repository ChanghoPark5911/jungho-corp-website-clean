## 🔧 구현 방법

### TraditionalLayout 컴포넌트 개선

현재 각 페이지에서 `sidebarItems`를 prop으로 전달하고 있지만, 이를 **자동화**할 수 있습니다:

```javascript
// TraditionalLayout.js 개선안
const TraditionalLayout = ({ children, category = 'home' }) => {
  // category에 따라 자동으로 사이드바 메뉴 생성
  const sidebarItems = getSidebarItemsByCategory(category);
  
  return (
    <div className="flex">
      <Sidebar items={sidebarItems} />
      <main>{children}</main>
    </div>
  );
};
```

### Category 정의
- `home`: Gateway 메뉴
- `about`: 회사소개 메뉴
- `subsidiaries`: 계열사 메뉴
- `media`: 미디어/PR 메뉴
- `support`: 고객지원 (사이드바 없음)

## 📝 권장 사항

### ✅ Classic & Hybrid 버전
1. **통일된 사이드바 원칙** 적용
   - 홈페이지: Gateway (4개 카테고리)
   - 카테고리별 페이지: 해당 카테고리 세부 메뉴

2. **TraditionalLayout 개선**
   - `category` prop 기반 자동 사이드바 생성
   - 각 페이지에서 일일이 `sidebarItems` 전달 불필요

3. **일관성 유지**
   - Classic과 Hybrid는 같은 사이드바 구조
   - 디자인만 다르게 (Classic: 심플, Hybrid: 현대적)

### ✅ V2 버전
- 현재 상태 유지 (MegaMenu만 사용)
- 좌측 사이드바 불필요
