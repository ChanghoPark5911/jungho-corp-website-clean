## 🎯 구현 방식

### 기본 원칙
```javascript
// 1. useI18n 훅 사용
const { t, currentLanguage } = useI18n();

// 2. 조건부 렌더링 (영어 우선, 한국어 fallback)
const text = currentLanguage === 'en' 
  ? (t('key.path') || adminData?.field || 'English Default')
  : (adminData?.field || '한국어 기본값');

// 3. 관리자 데이터와 i18n 통합
// - 영어: i18n 번역 > 관리자 데이터 > 하드코딩 영어
// - 한국어: 관리자 데이터 > 하드코딩 한국어
```

### 번역 키 구조
```javascript
{
  // 네비게이션
```

## ❓ 자주 묻는 질문 (FAQ)

### Q1: IRGS 섹션에 한국어가 남아있는데요?
**A**: 브라우저 캐시 문제일 수 있습니다. 다음을 시도해주세요:
1. 브라우저에서 `Ctrl+Shift+R` (Windows) 또는 `Cmd+Shift+R` (Mac)으로 강력 새로고침
2. 개발자 도구(F12) → Application → Local Storage → 해당 도메인 → Clear
3. 관리자 페이지에서 영어 번역 다시 저장

### Q2: 각 페이지마다 다국어를 적용해야 하나요?
**A**: 네, 각 페이지마다 개별적으로 적용이 필요합니다. 하지만 작업 순서는:
1. **홈페이지 (완료)**: 가장 많이 보는 페이지
2. **네비게이션 (완료)**: 모든 페이지에서 보이는 메뉴
3. **ABOUT 페이지**: 회사 소개 정보
4. **계열사/미디어/고객센터**: 순차적으로 진행

### Q3: 패밀리 사이트 링크는 어떻게 처리하나요?
**A**: 패밀리 사이트(Magic CLARUS, REDSSOCKSOO)는 **외부 사이트**이므로:
- 현재 홈페이지의 언어 설정과 무관하게 동작
- 각 외부 사이트가 자체적으로 언어를 관리
- **신경 쓰지 않아도 됩니다** ✅

### Q4: 모든 페이지를 지금 다 해야 하나요?
**A**: 아니요. 우선순위에 따라 단계적으로 진행:
1. ✅ 홈페이지 + 네비게이션 (완료)
2. 🔜 ABOUT 페이지들 (다음 단계)
3. 🔜 계열사/미디어/고객센터 (그 다음)
4. 🔜 Footer/기타 컴포넌트 (마지막)

### Q5: 관리자 페이지에서 어떻게 번역을 관리하나요?
**A**: 
1. 관리자 페이지 로그인 (`/admin`)
2. "다국어 관리" 탭 선택
3. 언어 선택 (English)
4. 섹션별로 번역 입력
5. "저장" 버튼 클릭
6. 홈페이지에서 언어 전환하여 확인

## 💡 권장사항

### 1. 번역 관리
- 번역은 **관리자 페이지**에서 일괄 관리
- 번역 키는 의미있는 계층 구조 사용
- 번역이 없으면 자동으로 한국어 표시 (Fallback)

### 2. 테스트
- 각 페이지별로 번역 적용 후 즉시 테스트
- 모바일/데스크톱 모두 확인
- 다크모드에서도 확인

### 3. 우선순위
- **홈페이지 먼저** (방문자가 가장 많이 보는 페이지)
- **네비게이션 먼저** (모든 페이지에서 보임)
- **핵심 페이지 먼저** (ABOUT, 계열사)
