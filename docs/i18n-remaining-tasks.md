## 🎯 작업 방식 (패턴)

모든 페이지에서 동일한 패턴으로 작업:

### 1단계: useI18n 훅 추가
```javascript
import { useI18n } from '../../hooks/useI18n';

const MyPage = () => {
  const { currentLanguage } = useI18n();
  // ...
}
```

### 2단계: 텍스트를 조건부 렌더링으로 변경
```javascript
// Before (하드코딩)
<h1>정호그룹</h1>

// After (다국어 지원)
<h1>{currentLanguage === 'en' ? 'Jungho Group' : '정호그룹'}</h1>
```

### 3단계: 복잡한 텍스트는 JSX 조건부 렌더링
```javascript
{currentLanguage === 'en' ? (
  <>
    English text here
    <br />
    Multiple lines
  </>
) : (
  <>
    한국어 텍스트
    <br />
    여러 줄
  </>
)}
```

## ⚠️ 현재 문제점 및 해결 방법

### 문제 1: "일관성 부족"
**증상**: 어떤 페이지는 영어로 나오고, 어떤 페이지는 한국어로만 나옴

**원인**: 
- React는 각 페이지/컴포넌트가 독립적으로 작동
- 각 페이지마다 개별적으로 다국어 코드를 추가해야 함
- 현재는 홈페이지, ABOUT 메인만 완료됨

**해결책**:
- 위의 Phase 1~5를 순차적으로 진행
- 각 페이지에 `useI18n` 훅 추가 및 조건부 렌더링 적용

---

### 문제 2: "LocalStorage 초기화 필요"
**증상**: 새로운 번역 키가 반영되지 않음

**해결책**:
```javascript
// 브라우저 콘솔에서 실행
localStorage.removeItem('i18nTranslations');
location.reload();
```

---

### 문제 3: "번역 우선순위"
**현재 상태**:
- 영어: i18n 번역 → 하드코딩 영어
- 한국어: 하드코딩 한국어

**권장 순서**:
1. 한국어 콘텐츠 완전히 확정
2. 전문 번역 의뢰
3. 관리자 페이지(`/admin`)에서 번역 입력
