## 1. 배포 개요

### 1.1 배포란?

배포(Deployment)란 개발 환경에서 작업한 홈페이지를 실제 사용자가 접속할 수 있는 서버에 올리는 과정입니다.

### 1.2 배포 방법 비교

| 방법 | 서버 | 장점 | 단점 |
|------|------|------|------|
| 가비아 호스팅 | 가비아 | 기존 서비스 활용, 이메일 영향 없음 | 수동 업로드 필요 |
| Vercel | Vercel | 자동 배포, 무료 | 새 서비스 추가 |

### 1.3 권장 방법

**가비아 호스팅 배포를 권장합니다.**
- 이미 비용을 지불하고 있는 서비스 활용
- 회사 이메일 서비스에 영향 없음
- DNS 설정 변경 불필요

## 3. 방법 1: 가비아 호스팅 배포 (권장)

### 3.1 Step 1: 프로젝트 빌드

**빌드란?**
개발용 소스 코드를 실제 서버에서 실행 가능한 파일로 변환하는 과정입니다.

**빌드 실행 방법:**

```
1. 명령 프롬프트(CMD) 또는 PowerShell 실행

2. 프로젝트 폴더로 이동:
   cd C:\Work\jungho-corp-website-clean

3. 빌드 명령 실행:
   npm run build

4. 빌드 완료 메시지 확인:
   "The build folder is ready to be deployed."
```

**빌드 결과:**
- `build` 폴더에 배포용 파일이 생성됩니다
- 이 폴더의 모든 파일을 서버에 업로드합니다

### 3.2 Step 2: FileZilla 설정

**FileZilla 다운로드 및 설치:**
```
1. https://filezilla-project.org 접속
2. "Download FileZilla Client" 클릭
3. 설치 파일 다운로드 및 실행
4. 설치 완료
```

**FTP 접속 정보 입력:**

```
FileZilla 상단 빠른 연결:

호스트(H): ftp.junghocorp.com (가비아에서 확인)
사용자명(U): your_ftp_id (가비아에서 확인)
비밀번호(P): your_ftp_password (가비아에서 확인)
포트(P): 21

[빠른 연결] 클릭
```

### 3.3 Step 3: 파일 업로드

**FileZilla 화면 구성:**

```
┌─────────────────────────────────────────────────────────┐
│ FileZilla                                               │
├─────────────────────────────────────────────────────────┤
│ 로컬 사이트 (내 컴퓨터)    │ 원격 사이트 (서버)          │
│                           │                             │
│ C:\Work\jungho-corp-      │ /public_html/               │
│   website-clean\build\    │                             │
│                           │                             │
│ ├── static\               │ ├── (기존 파일들)           │
│ ├── images\               │                             │
│ ├── data\                 │                             │
│ ├── index.html            │                             │
│ └── ...                   │                             │
│                           │                             │
└─────────────────────────────────────────────────────────┘
```

**업로드 순서:**

```
1. 왼쪽(로컬): build 폴더로 이동
   C:\Work\jungho-corp-website-clean\build

2. 오른쪽(원격): 웹 루트 폴더로 이동
   /public_html/ 또는 /www/

3. 기존 파일 백업 (선택사항)
   - 기존 파일들을 백업 폴더로 이동

4. build 폴더 안의 모든 파일 선택
   - Ctrl + A로 전체 선택

5. 오른쪽 원격 사이트로 드래그 앤 드롭
   - 또는 우클릭 > 업로드

6. 파일 전송 완료 대기
   - 하단 전송 상태 확인
```

**업로드할 파일 목록:**

| 파일/폴더 | 설명 |
|-----------|------|
| index.html | 메인 HTML 파일 |
| static/ | CSS, JavaScript 파일 |
| images/ | 이미지 파일들 |
| data/ | JSON 데이터 파일들 |
| manifest.json | PWA 설정 |
| favicon.ico | 파비콘 |
| robots.txt | 검색엔진 설정 |
| sitemap.xml | 사이트맵 |

## 4. 방법 2: Vercel 자동 배포

### 4.1 Vercel 배포 개요

Vercel은 GitHub 저장소와 연결되어 코드 변경 시 자동으로 배포됩니다.

```
코드 수정 → Git Commit → Git Push → Vercel 자동 배포
```

### 4.2 수동 배포 명령

```
1. 명령 프롬프트에서 프로젝트 폴더로 이동:
   cd C:\Work\jungho-corp-website-clean

2. 변경사항 커밋:
   git add -A
   git commit -m "콘텐츠 업데이트"

3. GitHub에 푸시:
   git push origin main

4. Vercel 자동 배포 대기 (약 1-2분)

5. 배포 URL 확인:
   https://jungho-corp-website-clean.vercel.app
```

### 4.3 배포 상태 확인

```
1. Vercel 대시보드 접속:
   https://vercel.com/dashboard

2. 프로젝트 선택

3. Deployments 탭에서 배포 상태 확인:
   - Building: 빌드 중
   - Ready: 배포 완료
   - Error: 오류 발생
```

## 5. 도메인 연결

### 5.1 현재 상태

| 항목 | 현재 상태 |
|------|-----------|
| 도메인 | junghocorp.com (가비아) |
| 호스팅 | 가비아 호스팅 사용 중 |
| 이메일 | 가비아 이메일 사용 중 |

### 5.2 가비아 호스팅 사용 시

도메인 연결 작업이 필요 없습니다.
기존 가비아 호스팅에 파일만 업로드하면 됩니다.

### 5.3 Vercel 사용 시 도메인 연결

**Vercel에서 도메인 추가:**
```
1. Vercel 대시보드 > 프로젝트 선택
2. Settings > Domains
3. junghocorp.com 입력 > Add
4. DNS 레코드 정보 확인
```

**가비아 DNS 설정:**
```
1. 가비아 로그인
2. My가비아 > 도메인 관리
3. DNS 설정 클릭
4. 레코드 추가:
   - A 레코드: @ → 76.76.21.21
   - CNAME 레코드: www → cname.vercel-dns.com
5. 저장
```

**주의: MX 레코드(이메일)는 절대 삭제하지 마세요!**

## 7. 문제 해결

### 7.1 빌드 오류

**증상:** npm run build 실행 시 오류 발생

**해결 방법:**
```
1. Node.js 버전 확인:
   node --version
   (v18 이상 권장)

2. 패키지 재설치:
   rm -rf node_modules
   npm install

3. 다시 빌드:
   npm run build
```

### 7.2 FTP 연결 실패

**증상:** FileZilla에서 서버 연결 안됨

**해결 방법:**
```
1. FTP 정보 재확인 (가비아 > 호스팅 > FTP 정보)
2. 방화벽 확인 (포트 21 허용)
3. 패시브 모드 시도:
   FileZilla > 편집 > 설정 > 연결 > FTP > 전송 모드 > 수동
```

### 7.3 페이지 404 오류

**증상:** 특정 페이지 접속 시 404 오류

**해결 방법:**
```
1. React Router 설정 확인

2. .htaccess 파일 생성 (가비아 호스팅용):
   파일 내용:
   ---
   <IfModule mod_rewrite.c>
     RewriteEngine On
     RewriteBase /
     RewriteRule ^index\.html$ - [L]
     RewriteCond %{REQUEST_FILENAME} !-f
     RewriteCond %{REQUEST_FILENAME} !-d
     RewriteRule . /index.html [L]
   </IfModule>
   ---

3. .htaccess 파일을 서버 루트에 업로드
```

### 7.4 이미지 로딩 실패

**증상:** 이미지가 깨져서 표시됨

**해결 방법:**
```
1. 이미지 파일 업로드 확인
2. 파일 경로 대소문자 확인 (.jpg vs .JPG)
3. 파일 권한 확인 (644)
4. 브라우저 캐시 삭제 후 재확인
```

### 7.5 HTTPS 오류

**증상:** 보안 경고 또는 혼합 콘텐츠 오류

**해결 방법:**
```
1. SSL 인증서 확인 (가비아에서 발급)
2. 모든 리소스가 https:// 로 로드되는지 확인
3. 개발자 도구(F12) > Console에서 오류 확인
```

## 8. 롤백 방법

### 8.1 가비아 호스팅 롤백

**이전 버전 백업이 있는 경우:**
```
1. FileZilla로 서버 접속
2. 현재 파일들 삭제 (또는 다른 폴더로 이동)
3. 백업 파일들 업로드
4. 웹사이트 확인
```

### 8.2 Vercel 롤백

**이전 배포 버전으로 롤백:**
```
1. Vercel 대시보드 접속
2. 프로젝트 > Deployments
3. 이전 배포 버전 찾기
4. "..." 메뉴 > "Promote to Production"
5. 확인
```

**Git으로 롤백:**
```
1. 이전 커밋 확인:
   git log --oneline

2. 롤백:
   git revert HEAD
   git push origin main

3. Vercel 자동 재배포 대기
```
